=========================================
JM-Monitor 需求驱动型数据流向草图 (V1.0)
=========================================

【 场景一：心跳数据怎么进来？】
外部访客 (点击API) 
   └───► 1. 敲击阿里云大门 (Nginx 80/443端口)
           └───► 2. 顺着管道转发 (proxy_pass)
                   └───► 3. 递交数据给 [jm-monitor 容器] (10000端口)
                           └───► 4. 写入内存账本 (更新时间戳)

-----------------------------------------

【 场景二：雷达怎么在内部掐死故障？】
[jm-monitor 容器] (内网高能雷达)
   └───► 1. 每隔10秒在走廊里大喊一声："order-service！" (内网DNS)
           ├───► [情况A：200 OK] ──► 邻居订单容器在线，一切正常，闭嘴。
           └───► [情况B：失联/报错] ─► 触发刚性状态机变更！
                                       └───► 2. 拼装飞书卡片 JSON
                                               └───► 3. 隔空打向外网 [飞书服务器]
                                                       └───► 4. JM手机弹窗报警 🚨
